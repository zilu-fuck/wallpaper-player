# Wallpaper Player 1.2.3 特殊版本开发日志

记录日期：2026-06-23
桌面端版本：`1.2.3`
手机端版本：`0.1.2`
发布页：<https://github.com/zilu-fuck/wallpaper-player/releases/tag/v1.2.3>

## 版本定位

`v1.2.3` 是一个特殊版本，核心目标不是单点修补，而是把“本地视频库”推进到“可理解、可管理、可跨端查看”的阶段。这个版本集中验证四条主线：

- 桌面端视频理解流程能否完整跑通。
- VLM 模型下载、检测、配置、启动、关闭能否尽量在项目设置页内完成。
- 手机端能否作为远程入口，请求电脑端执行分析，并查看电脑端返回的结果。
- 隐私目录能否在桌面端和手机远程库里都默认隐藏。

因为改动跨度比较大，这个版本被标记为特殊版本，建议先用于内部验证，再作为常规更新扩大分发。

## 本次完成的主要能力

### 1. 视频理解工作流

桌面端新增了视频分析能力，分析入口放在视频操作菜单中。点击分析后，主界面会打开右侧视频分析侧边栏，用来显示队列、进度、已有结果和分析完成后的卡片。

播放器内部也新增了分析结果展示区域。这里不再承担完整队列管理，而是更接近“当前视频的结果面板”：用户看视频时可以快速看到最新分析结果；完整的排队、进度、删除和弹窗查看，则放到主界面右侧栏处理。

分析结果采用“同一视频只保留最新结果”的策略。新的分析完成后会覆盖旧结果，避免一个视频出现多个结果文件导致前端不知道展示哪一个。结果文件命名保留视频名、分析结果标识和识别码，兼顾可读性和稳定匹配。

### 2. 视频分析侧边栏

新增桌面端视频分析侧边栏，承担分析任务的统一管理：

- 查看当前分析队列和进度。
- 查看当前目录下已有分析结果。
- 点击卡片弹窗查看完整结果。
- 支持单条删除分析结果。
- 支持多选后批量删除结果。
- 主界面新增入口，可主动打开右侧边栏。

这部分的重点是把“分析按钮”和“分析管理”拆开：视频卡片只负责发起分析或查看当前视频结果，主界面侧边栏负责整体状态。

### 3. VLM 配置一条龙

设置页新增了视频理解相关配置，并重新整理成更清晰的结构。默认情况下视频理解是关闭的；用户勾选开启后，后续参数才展开，避免设置页在默认状态下过于拥挤。

本次设置页覆盖了这些能力：

- 显示模型推荐目录和当前自定义目录。
- 支持用户自定义 VLM 模型目录和分析结果保存目录。
- 支持从 Hugging Face 预设模型列表中选择模型。
- 支持选择量化精度，默认按 `Q4_K_M` 路线处理。
- 支持点击按钮下载模型，并展示下载进度。
- 下载到指定目录后自动写入配置。
- 支持检测目录下已经存在的 VLM 模型，兼容用户从其他地方手动下载的模型。
- 支持在设置页直接启动、连接、停止本地 VLM 服务。
- 项目关闭时，会同步关闭由项目启动的 VLM 模型服务。

文本模型部分保留了两条路线：可以使用本地模型，也可以配置外接大模型 API。默认倾向使用本地子项目里的配置，用户需要时再切换到外接 API。

### 4. 视频理解子项目整理

`video comprehension` 子项目只保留和视频分析、输出、API、说明文档相关的部分，去掉了与当前播放器集成无关的冗余内容。

模型文件、输出文件、虚拟环境、缓存和本地配置没有纳入 git。打 Windows 包时，只把运行所需的最小子项目内容带入安装包，避免把大模型、结果文件和本机状态打进去。

### 5. 手机端视频分析入口

手机端新增视频分析能力，但分析本身仍由电脑端执行。手机端负责发起请求、查看进度和展示结果，电脑端负责模型、队列、分析和结果保存。

手机端本次新增：

- 视频播放页分析入口。
- 分析进度展示。
- 摘要、标签和时间线结果展示。
- 与电脑端远程服务交互的分析 API。
- 移动端 UI 调整，让分析面板在小屏幕上更容易阅读。

这个设计避免手机端承担本地大模型推理压力，也让同一个分析结果可以在电脑和手机之间复用。

### 6. 隐私目录模式

新增隐私目录模式。用户添加目录时可以选择设为隐私目录，也可以在侧栏目录菜单里后续切换。

隐私目录的行为：

- 默认不显示在桌面端侧栏中。
- 默认不出现在手机远程库中。
- 第一次使用时需要设置隐私密码。
- 后续显示隐私目录需要输入密码解锁。
- 解锁只在本次打开期间有效。
- 支持手动锁定隐私目录。
- 支持取消某个目录的隐私状态。
- 密码错误有次数限制，连续错误后会短暂锁定。

这次实现的目标不是做加密保险箱，而是提供一个“默认不暴露”的使用模式，降低普通浏览和手机远程访问时误展示私密目录的风险。

## 关键实现取舍

### 结果文件只保留最新

早期方案允许一个视频存在多个分析结果，但这会让前端展示逻辑变得混乱：视频内部“显示分析结果”按钮到底应该打开哪一个文件并不明确。

最终改为同一视频保留最新分析结果。这样用户重新分析时，新的结果会替代旧结果，视频卡片、播放器、侧边栏都能稳定指向同一份结果。

### VLM 管理放在播放器设置页

VLM 的下载、目录配置、检测、启动和连接都放进播放器设置页，而不是要求用户自己去子项目里手动操作。原因是这套能力面向普通用户时，最容易出问题的不是代码，而是模型路径、模型是否存在、服务是否启动、端口是否连得上。

设置页因此承担了“可视化模型管理器”的职责：告诉用户推荐目录在哪里、当前目录在哪里、能下载哪些模型、下载进度如何、目录下有没有可用模型、服务是否已经启动。

### 手机端只做远程入口

手机端不直接跑模型。分析任务由电脑端执行，结果再传给手机端。这样可以避免 Android 端模型体积、性能、显存/内存和后台任务管理问题，也让电脑端已有的分析队列和结果保存策略保持唯一来源。

### 隐私目录默认从远程库隐藏

隐私目录不仅在桌面侧栏隐藏，也从手机远程库中过滤。这个决策很重要，因为手机端远程访问本质上扩大了视频库可见范围。如果只在桌面隐藏，手机端仍能看到，就会破坏用户对“隐私目录”的直觉。

## 已知限制

- Android APK 当前使用 debug keystore 签名的 release 构建，只适合内部测试。
- 视频理解依赖用户配置的文本模型和 VLM 服务；模型不可用时，需要先在设置页完成下载、检测和启动。
- VLM 推理的实际速度、显存占用和稳定性取决于用户机器、模型大小、量化精度和 llama.cpp 后端能力。
- Hugging Face 模型下载依赖网络可达性，国内网络环境可能需要用户自行准备模型文件。
- Android 构建在中文路径下仍可能失败，后续建议保留 ASCII 构建目录策略，或进一步固化到脚本里。
- 当前隐私模式是目录隐藏和访问过滤，不是对视频文件本体进行加密。

## 后续建议

- 把 Android 的 ASCII 路径构建流程脚本化，避免每次手动复制工作区。
- 为 VLM 服务状态增加更明确的诊断信息，例如端口占用、模型缺失、GPU 后端不可用、下载中断恢复。
- 增加视频分析结果的导出入口，方便用户把摘要或时间线复制到其他工具。
- 为隐私目录补充更明显的设置入口，例如集中管理所有隐私目录。
- 做一次真实长视频分析压力测试，记录不同模型、不同抽帧策略、不同上下文长度下的耗时和结果质量。
