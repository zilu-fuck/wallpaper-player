## 版本定位

`v1.2.5` 是一次桌面端和手机端同步发布版本：桌面端升级到 `1.2.5`，Android 手机端升级到 `0.1.3`。这版重点把视频库从“每次扫描目录”推进到“可缓存、可增量刷新、可跨端筛选和管理”的状态。

## 更新内容

- 新增视频元数据缓存：电脑端会缓存时长、分辨率、帧率、编码和容器信息，并在扫描和手机端视频库接口中复用。
- 新增扫描索引：视频目录扫描结果会持久化到用户数据目录，后续打开可先用索引快速出列表，再后台刷新完整目录。
- 新增 Wallpaper Engine 元数据增强：读取 `project.json` 中的系统标签，并识别 `R18` / NSFW 等成人内容标记。
- 新增桌面端和手机端多标签交集筛选：同时勾选多个标签时，只显示同时命中的视频。
- 新增 VLM 分析标签绑定：视频理解返回的标签可以由用户确认后加入视频自定义标签。
- 新增手机端播放背景设置：默认黑色背景，也可在设置中切换为封面背景。
- 新增手机端未连接电脑前的更新入口：绑定前也能查看当前版本并检查 APK 更新。
- 新增手机端转码队列和缓存管理：不兼容格式可排队生成 H.264/AAC MP4，并支持列出任务、取消任务、清理缓存。
- 优化桌面端播放器进度缩略图：缩略图从控制条下方浮出，三角指向进度条，并补充边界约束避免跑出窗口。
- 优化桌面端设置页、播放器、远程服务拆分：设置页和播放器按区块拆组件，远程服务拆分为 library、media、tags、analysis、transcode 等 handler。
- 修复更新提示无法正确渲染 HTML/Markdown release notes 的问题。
- 修复视频排序：桌面端和手机端都使用中文自然排序，并增加稳定兜底字段。
- 修复扫描索引刷新状态：索引命中时会向客户端暴露 `refreshing/indexed`，并在后台刷新完成后自动补拉最新结果。
- 修复扫描索引文件名过长风险：索引文件名改为固定长度 `sha256`。
- 修复手机端更新误报：手机端现在从 Release APK 文件名解析手机版本，不再把桌面端 `v1.2.5` 标签当作手机端版本比较。
- 修复转码队列竞态：同一视频同一质量的并发请求会复用同一个任务，并在源文件更新后丢弃旧 ready 缓存重新转码。
- 修复远程 JSON 请求体错误状态：无效 JSON 返回 400，过大请求体返回 413。
- 防止本机 `settings.json` 被误提交。

## 安装包

- Windows 安装包：`Wallpaper-Player-Setup-1.2.5.exe`
- Windows 便携版：`Wallpaper-Player-1.2.5.exe`
- Android APK：`wallpaper-player-mobile-0.1.3-arm64-release.apk`

## 验证

- `npm.cmd run build`
- `npm.cmd run verify:remote-library`
- `npm.cmd run verify:remote-api`
- `npm.cmd run verify:remote-mobile`
- `npm.cmd run verify:mobile-player`
- `npm.cmd run verify:video-metadata`
- `npm.cmd run verify:mobile-transcode`
- `npm.cmd run verify:mobile-transcode-concurrency`
- `npm.cmd run verify:mobile-multi-device`
- `cd mobile && npm.cmd run typecheck`

## 注意事项

- Android APK 仍使用 debug keystore 签名的 release 构建，仅适合内部测试。
- 手机端播放原始视频仍依赖系统解码能力，不兼容源文件建议使用兼容转码。
- 自动更新依赖 GitHub Releases 网络可达性，无法访问时可手动分发 Windows 安装包和 Android APK。
