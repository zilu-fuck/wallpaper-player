# Wallpaper Player v1.4.0

这是一次资源入口和下载体系的大版本更新：本地目录、网络资源、磁链下载被拆成更清晰的入口，并新增下载中心。

## 新增

### 添加资源入口

- “添加资源”拆分为本地资源、网络资源、磁链资源。
- 本地资源继续接入原有目录扫描流程。
- 网络资源进入独立的“网络资源库”，不再挤在目录扫描器里。

### 网络资源库

- 支持保存视频直链、HLS/DASH 链接、NAS/局域网链接和网页入口。
- 支持网络资源搜索、列表/网格视图、编辑名称和地址、单个删除、批量删除。
- 支持网页资源的剧集展开，已解析剧集可直接切换播放。
- 对无法直接解析的视频网站，保存为“网页入口”，用内置网页打开观看。

### 网页资源解析

- 新增网页资源解析框架，当前包含站点解析、通用媒体地址提取和 yt-dlp 兜底。
- 支持 Bilibili、X/Twitter、部分网页播放器和通用直出媒体地址。
- yt-dlp 会跟随本机代理环境；需要浏览器 Cookie 的站点可后续通过环境变量启用。

### 下载中心

- 新增独立“下载中心”，采用类似迅雷的任务列表和右侧详情结构。
- 内置 aria2c 作为下载引擎，支持 HTTP/HTTPS 和 BT 磁链。
- 磁链会先选择保存目录，再等待元数据，解析出文件列表后默认勾选视频文件和字幕文件。
- 开启 tracker、DHT、DHT6、PEX、LSD、端口检测和资源状态提示。
- 默认完成后停止做种。
- 下载完成后自动刷新保存目录；未加入视频库的保存目录可在下载中心加入库或手动扫描。

### 迅雷接管

- 下载中心新增“迅雷接管”入口。
- 检测到本机迅雷时可直接拉起迅雷；未检测到时提示安装。
- 应用会记录用户选择的目录，便于下载完成后扫描或加入视频库。
- 迅雷实际保存位置需在迅雷中确认，详细速度和进度也在迅雷中查看。

### 便携版与移动版同步

- Windows 安装版和便携版使用同一套 1.4 桌面功能与内置依赖，均包含网络资源、下载中心、aria2、yt-dlp 和迅雷接管入口。
- Android 移动版更新到 0.1.4，可在手机端看到电脑端已添加的网络资源，并通过电脑端代理播放直链、已解析网页视频和 HLS 分片。
- 手机端网络资源支持收藏、标签和“电脑播放”；添加资源、网页兜底观看、下载中心、磁链选择、aria2 和迅雷接管仍由电脑端负责。

## 改进

- 设置页新增下载引擎状态，包括 aria2、yt-dlp、BT 端口、tracker 和迅雷检测信息。
- 网络播放和下载的主进程边界收紧：播放/下载只信任已加入网络资源库的地址或其剧集地址。
- 内置网页默认拒绝第三方页面的摄像头、麦克风、定位、通知等系统权限请求。
- 网页解析增加响应大小限制，避免异常网页拖慢主进程。
- 第三方声明补充 aria2 和 yt-dlp 的许可证、来源与校验信息。

## 已知限制

- DRM 网站通常只能以内置网页方式观看，不能提取为可下载视频地址。
- m3u8/mpd 当前支持播放，完整离线打包下载仍需后续接入 HLS/DASH 下载流程。
- aria2 对 BT v2 / btmh / 64 位 hash 的兼容有限，本版优先支持 BT v1 / btih 磁链。
- 冷门磁链可能长时间拿不到元数据，取决于 tracker、DHT 和资源热度。
- 迅雷接管任务的详细进度由迅雷管理，本应用只记录入口和保存目录用于后续扫描。
- 移动端不内置下载引擎，也不直接管理电脑端保存目录；网络资源和下载任务的新增/删除仍需在电脑端操作。
