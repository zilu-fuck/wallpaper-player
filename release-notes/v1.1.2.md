## 更新内容

- 新增应用内更新提示：检测到远端新版本后，会显示目标版本号、更新说明和操作按钮。
- 自动更新改为完整确认流程：发现更新后由用户点击下载，下载过程中显示进度，下载完成后可点击重启安装。
- 修复更新已发现但界面没有任何提示的问题，安装版用户现在可以在应用内完成发现、下载、安装闭环。
- 优化目录切换体验：近期扫描结果会缓存，并通过目录监听与 TTL 兜底失效，减少切换目录时的重复全量扫描。
- 将手动检查更新入口移动到设置面板，移除右下角常驻检查按钮。
- Windows 安装器完成页新增“创建桌面快捷方式”选项，安装结束时可由用户选择是否创建。
- 改进 mpv 启动失败时的错误提示，mpv 进程提前退出时会带出更具体的输出信息。
- 发布流程现在会读取版本对应的双语更新报告，并写入 GitHub Release 正文，方便自动更新面板展示更新内容。

## Update Notes

- Added an in-app update notice that shows the target version, release notes, and update actions when a remote version is found.
- Changed auto updates to a full user-confirmed flow: check, download with progress, then restart to install.
- Fixed the case where an update could be detected without any visible prompt in the app.
- Improved folder switching by caching recent scan results with directory watcher invalidation and a TTL fallback.
- Moved the manual update check entry into Settings and removed the always-visible bottom-right check button.
- Added a Windows installer finish-page option for creating a desktop shortcut after installation.
- Improved mpv startup error reporting by surfacing recent mpv output when the process exits early.
- Release builds now publish versioned bilingual release notes to the GitHub Release body, so the app can display the same notes during update checks.

## 自动更新说明

- 安装版会通过 GitHub Releases 的 `latest.yml` 检测新版本。
- 便携版不会自动更新，请到 Releases 页面手动下载新版。
- 从旧版升级时，请使用 `Wallpaper-Player-Setup-1.1.2.exe` 安装包。
- 覆盖后的 `v1.1.2` 安装包仍使用相同版本号；已安装 `1.1.2` 的客户端不会因为同版本覆盖而再次触发自动更新。

## Auto Update Notes

- The installer build checks GitHub Releases via `latest.yml`.
- The portable build does not auto-update. Please download new portable builds manually from Releases.
- To upgrade from an older installed version, use `Wallpaper-Player-Setup-1.1.2.exe`.
- The overwritten `v1.1.2` installer keeps the same version number, so already-installed `1.1.2` clients will not receive another auto-update for this same-version replacement.
